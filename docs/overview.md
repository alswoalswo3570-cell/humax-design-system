# UX Guide Overview

Welcome to the UX Guide. This documentation provides the foundational design tokens, UI components, and usage guidelines for building consistent, accessible, and beautiful products.

## Purpose
This guide serves as a machine-readable, implementation-friendly design system reference that helps humans and AI generate consistent product UI.

## Audience
- Product Managers
- Designers
- Frontend Developers
- AI Coding Agents

## Core Principles
1. **Machine-readable first**: Structured definitions (JSON) that AI can read and reuse.
2. **Semantic naming**: Use semantic names (`color.text.primary`) instead of raw visual names (`blue500`).
3. **Documentation + implementation alignment**: Tokens, rules, and preview examples stay aligned.
4. **Minimal but scalable**: Start small, structure for scale.
