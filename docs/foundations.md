# Foundations

Design tokens are the visual design atoms of the design system — specifically, they are named entities that store visual design attributes. We use them in place of hard-coded values (such as hex values for color or pixel values for spacing) in order to maintain a scalable and consistent visual system.

## Categories

### Colors
Semantic color tokens for backgrounds, text, borders, and interactive states.
- `color.background.*`
- `color.text.*`
- `color.border.*`
- `color.action.*`

### Spacing
A consistent scale for padding, margins, and gaps.
- `space.xs` (4px)
- `space.sm` (8px)
- `space.md` (16px)
- `space.lg` (24px)

### Radius
Standardized border radii for components.
- `radius.sm` (2px)
- `radius.md` (6px)
- `radius.lg` (8px)
- `radius.full` (9999px)

### Typography
Font families, sizes, weights, and line heights.
- `typography.fontFamily.sans`
- `typography.fontSize.base`
- `typography.fontWeight.medium`
